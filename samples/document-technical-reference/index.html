Technical Implementation Reference Document

Process document intended to provide application design rationale and implementation details for developers. Contrast, repetition, alignment, and proximity design principles were employed for a visually pleasing document with intuitive navigation.

Description#

Document Design

Document Design Principles

Through a custom style set with a branded color scheme, document elements are differentiated by font size, color, case, spacing, and adornments such as borders and background color. White space is used for separating sections in a reference document designed to be skimmed.

Extensive Heading Structure

The section heading structure provides document organization so the reader can skim for pertinent information. The heading structure also provides targets for internal navigation.

Illustrative Figures

Figures are implemented with a consistent background and font face to distinguish from surrounding content. Figures and tables are accompanied by captions used to generate the lists of figures and tables included in the document appendix.

External Content References

The document appendix includes bookmarks that contain short descriptions of related information and links to external content.
